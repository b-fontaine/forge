# Forge Reference Examples

<!-- Audit: C.1 (c1-reference-project) -->

This directory holds **curated reference projects** demonstrating the
Forge framework on each archetype. Each example is a fully-scaffolded
project tree generated by the actual `forge init` command, then
committed verbatim. Adopters can inspect the artefacts (proposals,
specs, designs, ADRs, tasks, archived demo changes) directly on
GitHub without cloning.

Future archetypes (mobile-only B.4, flutter-firebase B.2) will add
sibling directories here. The directory layout is flat — one example
per archetype, no nesting.

## Why these examples are excluded from Forge's own gates

Forge's deterministic gates (`verify.sh`,
`constitution-linter.sh`) ship with **skip-guards** (FR-GL-026,
FR-GL-027) that detect when they run from the Forge framework
repository and exclude the `examples/` subtree. Each example owns
its own gates and runs them via the example's local
`.forge/scripts/verify.sh`.

When a PR touches `examples/**`, the framework's CI workflow
(`forge-ci.yml`) runs a dedicated `example` job that invokes the
example tree's gates. PRs that don't touch `examples/**` skip the
job entirely (paths-filter via `dorny/paths-filter@v3`).

## How an example was built

Every example tree was generated by running the actual archetype
scaffolder once on a clean tmpdir, then committed verbatim. See the
example's own `README.md` for the exact reproduction commands.
